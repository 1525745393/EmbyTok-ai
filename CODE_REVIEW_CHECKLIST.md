# EmbyTok 代码审查清单

## 📋 通用审查清单

### 🔴 必须检查（阻塞级别）

- [ ] **构建成功**：`npm run build` 无错误
- [ ] **类型安全**：无 TypeScript 类型错误
- [ ] **无测试失败**：`npm run test:run` 全部通过
- [ ] **无明显 Bug**：代码逻辑正确，无明显功能问题
- [ ] **安全性**：无安全漏洞

### 🟡 重要检查

- [ ] **代码风格一致**：遵循项目规范
- [ ] **命名清晰**：变量、函数命名有意义
- [ ] **单一职责**：函数/组件职责单一
- [ ] **错误处理**：异常情况正确处理
- [ ] **性能考虑**：无明显性能问题
- [ ] **文档更新**：必要的文档已更新
- [ ] **测试覆盖**：新功能有相应测试

### 🟢 可选优化

- [ ] **代码简化**：有无更简洁的写法
- [ ] **可复用性**：有无可提取的公共逻辑
- [ ] **注释完善**：复杂逻辑有说明

---

## 🔧 专项审查清单

### React 组件审查

- [ ] 组件命名为 PascalCase
- [ ] Props 有完整类型定义
- [ ] 避免不必要的重渲染（合理使用 memo、useMemo、useCallback）
- [ ] Hook 使用正确（依赖项完整）
- [ ] Effect 有清理函数（如有必要）
- [ ] 避免 Prop Drilling（考虑使用 Context）
- [ ] 列表渲染有稳定的 key

### TypeScript 审查

- [ ] 避免使用 any，优先使用 unknown
- [ ] 函数参数和返回值有类型
- [ ] 对象类型定义完整
- [ ] 使用可选链和空值合并
- [ ] 类型守卫使用合理（如有需要）

### 工具函数/服务审查

- [ ] 函数有 JSDoc 注释
- [ ] 有单元测试
- [ ] 边界条件处理
- [ ] 无副作用（纯函数优先）

### 安全审查

- [ ] 用户输入已验证
- [ ] 敏感信息不硬编码
- [ ] 错误信息不泄露内部细节
- [ ] 避免 XSS 风险

---

## 🎯 快速审查指南

| 变更类型     | 重点关注                           |
| ------------ | ---------------------------------- |
| **新功能**   | 功能完整性、设计合理性、测试覆盖   |
| **Bug 修复** | 问题根源分析、修复正确性、防止回归 |
| **重构**     | 等价性保证、测试通过、性能无退化   |
| **性能优化** | 性能数据对比、不影响可读性         |
| **文档更新** | 准确性、完整性                     |
