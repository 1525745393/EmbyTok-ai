# AGENTS.md - AI 代理角色与职责

本文档描述了在 EmbyTok 项目开发过程中参与的 AI 代理角色及其职责。

## 项目概述

EmbyTok 是一个为 Emby 和 Plex 媒体服务器设计的竖屏视频浏览客户端，提供类似 TikTok 的体验，让用户能够以更现代、便捷的方式浏览个人媒体库。

## AI 代理角色

### 1. 代码开发代理 (Code Development Agent)

**职责：**

- 实现新功能和修复 bug
- 重构代码以提高可维护性和性能
- 编写 TypeScript/React 组件
- 实现 Emby 和 Plex 媒体客户端的 API 集成
- 确保代码符合项目规范和最佳实践

**技术栈：**

- React 18
- TypeScript
- Vite
- Tailwind CSS
- Capacitor
- Lucide React

### 2. 测试代理 (Testing Agent)

**职责：**

- 编写单元测试和集成测试
- 执行代码质量检查
- 确保所有功能正常工作
- 验证构建过程的完整性

**测试工具：**

- Vitest
- 自定义测试设置

### 3. 文档代理 (Documentation Agent)

**职责：**

- 维护和更新项目文档
- 为新功能编写文档
- 确保文档清晰、准确、及时更新
- 协助用户理解项目的使用方法

**文档范围：**

- README 文档
- API 文档
- 部署指南
- 贡献指南

### 4. 设计代理 (Design Agent)

**职责：**

- 确保 UI/UX 设计的一致性
- 优化移动端和桌面端的用户体验
- 提供美观且实用的界面设计建议
- 确保设计与技术实现的协调

**关注要点：**

- 响应式设计
- 移动端优化
- TV 模式适配
- 整体视觉风格

### 5. 性能优化代理 (Performance Optimization Agent)

**职责：**

- 识别和修复性能瓶颈
- 优化渲染和加载性能
- 确保应用的流畅运行
- 进行代码效率改进

**优化方向：**

- 视频预加载
- 图片懒加载
- 内存管理
- 计算优化

## 协作原则

所有 AI 代理在协作时应遵循以下原则：

1. **小步重构**：每次只做一个小改动，然后测试。频繁提交，保持代码随时可工作。
2. **测试保障**：重构前确保有足够的测试。每次修改后运行测试，确保行为不变。
3. **代码审查**：重构后进行代码审查，确保质量。
4. **内存优化**：避免不必要的对象创建，及时释放不再需要的资源，注意内存泄漏问题。
5. **计算优化**：避免重复计算，使用适当的数据结构和算法，延迟计算直到必要时。
6. **命名约定**：使用有意义的、描述性的名称，遵循项目或语言的命名规范。
