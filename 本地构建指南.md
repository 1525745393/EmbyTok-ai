## 在你本地环境构建的完整步骤

### 第一步：准备环境

#### 1. 确保已安装以下软件

- [Node.js](https://nodejs.org/) v18 或更高
- [Java JDK](https://adoptium.net/) v17 或更高
- [Android Studio](https://developer.android.com/studio)（推荐，最简单）
- Git

#### 2. 克隆或拉取最新代码

```bash
cd 你的项目目录
git pull origin main
```

---

### 第二步：使用自动构建脚本（最简单）

#### 1. Linux/Mac 用户

```bash
# 进入项目目录
cd EmbyTok

# 运行自动构建脚本
./build-android.sh
```

#### 2. Windows 用户

如果你在 Windows 上：

- 方式一：使用 WSL (Windows Subsystem for Linux) 运行上面的脚本
- 方式二：使用 Git Bash 运行脚本
- 方式三：使用 Android Studio 手动构建（参考下文）

---

### 第三步：使用 Android Studio 手动构建（最稳）

#### 详细步骤：

1. 打开命令行，进入项目目录，执行：

   ```bash
   npm run build
   npx cap sync android
   ```

2. 打开 **Android Studio**
3. 选择 **Open an Existing Project**
4. 选择项目中的 `android` 文件夹
5. 等待 Gradle 同步（第一次需要下载依赖，时间较长）
6. 同步完成后，在顶部菜单选择：
   **Build** → **Build Bundle(s) / APK(s)** → **Build APK(s)**

7. 等待构建完成！

#### APK 位置：

构建成功后，APK 文件在这里：

```
android/app/build/outputs/apk/debug/app-debug.apk
```

---

### 第四步：安装到手机测试

#### 方式一：使用 ADB（命令行）

```bash
# 确保手机连接并开启调试模式
adb devices

# 安装 APK
adb install android/app/build/outputs/apk/debug/app-debug.apk
```

#### 方式二：直接传输 APK 到手机

1. 将 APK 文件复制到手机
2. 在手机上打开 APK 文件安装
3. 允许“安装未知来源应用”

## 常见问题解决

### 1. Gradle 下载很慢或超时

✅ 已解决：已配置阿里云镜像源，无需额外操作！

### 2. 提示 Java 版本不匹配

确保安装了 JDK 17 或更高版本。
在 Android Studio 中可以设置：
**File** → **Settings** → **Build, Execution, Deployment** → **Build Tools** → **Gradle** → **Gradle JDK**

### 3. 提示 SDK 缺失

打开 Android Studio，它会自动提示安装缺失的 SDK 组件。
